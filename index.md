# index00.bin index01.bin index02.bin

这三个文件的结构完全相同。根据目前对文件最后修改时间的观察，猜测`index00.bin`记录着最新的目录信息。开机时会把`index00.bin`备份至`index02.bin`。关机前，会把`index00.bin`备份到`index01.bin`。

即，取出SD卡用电脑读取时，`index00.bin`与`index01.bin`的内容相同，而`index02.bin`中存储的是过时的数据。因此，后续分析都是对`index00.bin`的分析。

index文件总体结构大致如下：

| 段数 | 作用 |
| ---- | ---- |
| 1 | 总体信息 |
| 2 | 每一个`hivXXXXX.mp4`记录文件总体信息 |
| 3 | 不明 |
| 4 | 每一个`hivXXXXX.mp4`记录文件中每一段视频/抓拍的信息 |

## 第一段

本段位置固定为`0x0 - 0x3af`。在这一段中，按照长度`0x30`的内容为一节，分为19节。结构大致如下：

| 节数 | 作用 |
| ---- | ---- |
| 1 | 记录文件数量 |
| 2 | 当前写入的记录文件 |
| 3 | 记录抓拍的记录文件 |
| 4-19 | 空白 |

### 第1节 0x0 - 0x29

此节主要记录SD卡中共有多少个`hivXXXXX.mp4`记录文件。

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 4 | uint32 | 开箱后所有记录过的视频文件数量 |
| 8 | NA | 不明 |
| 2 | uint16 | SD卡目录中`hivXXXXX.mp4`文件数量 |
| 10 | NA | 不明 |
| 2 | NA | 不明，总是`0xFF 0xFF` |
| 22 | NA | 不明，总是`0x00` |

### 第2节 0x30 - 0x59

此节主要记录关机前正在写入的`hivXXXXX.mp4`文件编号`XXXXX`。

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 2 | uint16 | 关机时正在写入的视频文件编号`XXXXX` |
| 2 | NA | 不明，总是`00 00` |
| 4 | uint32 | 时间戳，该视频的开始时间 |
| 4 | uint32 | 时间戳，该视频的结束时间（即关机时间） |
| 12 | NA | 不明，总是`0x00` |
| 2 | NA | 不明，总是`0xFF 0xFF` |
| 22 | NA | 不明，总是`0x00` |

### 第3节 0x60 - 0x89

抓拍的照片会单独保存在一个`hivXXXXX.mp4`文件中。此节主要该抓拍记录文件的信息。

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 2 | uint16 | 用于记录抓拍的记录文件编号`XXXXX` |
| 2 | uint16 | 拍照数量 |
| 4 | uint | 时间戳，开箱后首次抓拍时间 |
| 4 | uint | 时间戳，最后一次抓拍的时间 |
| 4 | NA | 不明 |
| 2 | uint16 | 猜测为该文件支持的最大拍照数量 |
| 6 | NA | 不明，总是`0x00` |
| 2 | NA | 不明，总是`0xFF 0xFF` |
| 22 | NA | 不明，总是`0x00` |

### 第4-19节

这些节似乎没有被用到。

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 2 | NA | 不明，总是`0xFF 0xFF` |
| 22 | NA | 不明，总是`0x00` |
| 2 | NA | 不明，总是`0xFF 0xFF` |
| 22 | NA | 不明，总是`0x00` |

## 第三段

在这一段中，按照长度`0x20`为一节。每一节都对应了一个`hivXXXXX.mp4`记录文件。因为视频帧率、分辨率、HDR、SD卡容量等设置不同导致`hivXXXXX.mp4`记录文件数量也会不同。因此这一段的长度不固定。

例如，有234个`hivXXXXX.mp4`记录文件时，本段位置为0x500 - 0x223f。

### <span id="jump1">视频记录文件</span>

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 4 | uint32 | 对应的`hivXXXXX.mp4`文件序号`XXXXX` |
| 2 | uint16 | 0：未写入完成（如正在使用）<br/>1：写入完成 |
| 2 | uint16 | 文件中包含的视频片段数量-1 |
| 4 | uint32 | 时间戳，该视频文件的开始时间 |
| 4 | uint32 | 时间戳，该视频文件的结束时间 |
| 2 | uint16 | 此文件用来记录<br/>0：正常行车视频<br/>1：紧急录像<br/>2：当前正在写入的文件/抓拍文件 |
| 2 | NA | 不明 |
| 4 | NA | 不明，总是`0x00` |
| 4 | uint32 | 0x0：不明，常见<br/>0x2：不明，仅在紧急录像中出现<br/>0x64：按下紧急录像按钮开始的紧急录像 |
| 4 | NA | 不明，总是`0x00` |

## 第四段

在这一段中，按照长度`0x5000`为一节。同样地，每一节对应了一个`hivXXXXX.mp4`记录文件。

本节长度与位置不固定。例如有234个记录文件时，本节位置为0x2240 - 0x49423f

### 视频记录文件

对应视频记录文件时，每一节内可按照长度0x50的内容为一组，对应了视频文件中的某一段录像的内容。

一个视频文件分为几段录像是因为某一文件的记录范围内可能多次开关机，有多个紧急录像，或者进入/离开停车监控模式。

对长度0x50的每一组：

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 1 | uint8 | 0：视频记录文件<br/>2：抓拍（不存在于视频记录文件中） |
| 1 | NA | 不明，通常为`0x00`，偶尔为`0x01` |
| 2 | NA | 不明，总是`0x00` |
| 4 | uint32 | 0x13：普通行车记录<br/>0x00：停车监控 |
| 4 | uint32 | 时间戳，视频片段开始时间 |
| 4 | NA | 不明，总是`0x00` |
| 4 | uint32 | 时间戳，视频片段结束时间 |
| 20 | NA | 不明，总是`0x00` |
| 4 | uint32 | 本片段在`hivXXXXX.mp4`文件中的开始位置（含）<br/>视频片段的开始位置总是与0x10000对齐 |
| 4 | uint32 | 本片段在`hivXXXXX.mp4`文件中的结束位置（不含） |
| 1 | NA | 不明，猜测<br/>0：本段自然结束<br/>1：本段因关机而结束 |
| 1 | uint8 | 录像帧率（fps）<br/>30：行车记录（可能因设置而不同）<br/>1：停车监控 |
| 1 | NA | 不明，猜测<br/>0x0：行车记录<br/>0x63：停车监控 |
| 1 | NA | 不明，总是`0x01` |
| 4 | NA | 不明，总是`0x00` |
| 4 | NA | 不明，通常为`0x00`，偶尔为`0x64`或`0x02` |
| 12 | NA | 不明，总是`0x00` |
| 4 | NA | 不明，总是`0x00 0x00 0x00 0x10` |
| 4 | NA | 不明，总是`0x00 0x00 0x00 0x10` |

[在index文件第三段中提到过](#jump1)，第三段中记录了每一个视频文件中有几段录像（“文件中包含的视频片段数量-1”），但是此处的0x50长度的内容的数量可能多于前面的记录。这是由于对应的视频文件之前曾经记录过更多视频片段导致的。

例如，一个视频文件曾经记录过5段视频，则此处会留下5段记录。当这个视频文件被循环录像覆盖后，仅用来记录了1段录像。这时，index文件的此处第一组0x50长度的信息为最新1段覆盖后的视频的信息。之后的4组为已经被覆盖过的无效信息，应无视掉。

### 抓拍记录文件

与视频记录文件对应的结构类似，仍然是以0x50为一组，但此处每一组对应一张抓拍图片。

对0x50长度的每一组：

| 字节数 | 类型 | 猜测含义 |
| ---- | ---- | ---- |
| 1 | uint8 | 0：视频记录文件（不存在于抓拍记录文件中）<br/>2：抓拍 |
| 7 | NA | 不明，总是`0x00` |
| 4 | uint32 | 时间戳，拍照时间 |
| 4 | NA | 不明，总是`0x00` |
| 4 | uint32 | 时间戳，猜测为保存时间，通常与拍照时间相同 |
| 20 | NA | 不明，总是`0x00` |
| 4 | uint32 | 本片段在`hivXXXXX.mp4`文件中的开始位置（含）<br/>视频片段的开始位置总是与0x10000对齐 |
| 4 | uint32 | 本片段在`hivXXXXX.mp4`文件中的结束位置（不含） |
| 1 | NA | 不明，通常是`0x01`，也可能是`0x00` |
| 7 | NA | 不明，总是`0x00` |
| 1 | NA | 不明，通常是`0x67`，也可能是`0x03` |
| 15 | NA | 不明，总是`0x00` |
| 4 | NA | 不明，总是`0x00 0x00 0x00 0x10` |
| 4 | NA | 不明，总是`0x00 0x00 0x00 0x10` |
